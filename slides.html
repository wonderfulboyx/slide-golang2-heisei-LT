平成(Go1)までのエラーハンドリング

2つの文字列をIntに変換して足すコード

func addStrAsInt(a, b string) (int, error) {
	x, _ := strconv.Atoi(a)
	y, _ := strconv.Atoi(b)
	return x + y, nil
}

エラーハンドリングしようとすると…

func addStrAsInt(a, b string) (int, error) {
	x, err := strconv.Atoi(a)
	if err != nil {
		return 0, err
	}
	y, err := strconv.Atoi(b)
	if err != nil {
		return 0, err
	}
	return x + y, nil
}

Goに例外機構はない

  • throw/try/catchみたいな便利なものはない
    • じゃあどうするか
      - エラーをreturnして処理してもらう

x, err := strconv.Atoi(a)

  return err
}

いいところ

  • 可読性が高い=読めば何やってるかわかる

      • どこでミスったかわかりやすい
      • レビューしてて読みやすい

  • 冗長

令和(Go2)のエラーハンドリング

Go2ではcheck式 / handle構文 が導入されます！！(多分)

見た目の比較

Go1

func addStrAsInt(a, b string) (int, error) {
	x, err := strconv.Atoi(a)
	if err != nil {
		return 0, err
	}
	y, err := strconv.Atoi(b)
	if err != nil {
		return 0, err
	}
	return x + y, nil
}

Go2

func addStrAsInt(a, b string) (int, error) {
	handle err { return 0, err }
	x := check strconv.Atoi(a)
	y := check strconv.Atoi(b)
	return x + y, nil
}

🎉

すごい!!めちゃスッキリしてる!!

func addStrAsInt(a, b string) (int, error) {
	handle err { return 0, err }
	x := check strconv.Atoi(a)
	y := check strconv.Atoi(b)
	return x + y, nil
}

どう動くのか

末尾のerrorがnilかどうか判定してくれる

  • これだけで 
    handle err { return 0. err }
x := check strconv.Atoi(a)
  • 以下に相当することをやってくれる 
    x, err := strconv.Atoi(a)
if err != nil {
  return <handle構文内の処理>
}

もしhandleが複数あったら？

func process(user string, files chan string) (n int, err error) {
    handle err { return 0, fmt.Errorf("process: %v", err)  }      // handler A
    for i := 0; i < 3; i++ {
        handle err { err = fmt.Errorf("attempt %d: %v", i, err) } // handler B
        handle err { err = moreWrapping(err) }                    // handler C

        check do(something())  // check 1: C, B, A の順にhandleが評価される
    }
    check do(somethingElse())  // check 2: Aのhandleが評価される
}

さらに嬉しいところ

checkは式なので評価結果を直接関数に渡せる

これが

x, err := strconv.Atoi(a)
if err != nil {
	return 0, err
}
y, err := strconv.Atoi(b)
if err != nil {
	return 0, err
}
add(x, y) // intを2つとる関数


// こうは書けない！！
// add(strconv.Atoi(a), strconv.Atoi(b))

こうなる！

handle err { return 0, err }
add(check strconv.Atoi(a), check strconv.Atoi(b))

簡潔！！
